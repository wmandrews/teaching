CHARTS

Datawrapper

Datawrapper.de

This open source charting tool is a great way to quickly publish interactive charts online. All it requires is a chunk of data in spreadsheet format, and then you can choose the form and other options for the chart. Datawrapper includes an ability to save your charts to a library so you can easily access and edit them at a later date.

Chart types: Bar, column, line, pie, table.

GitHub Gists and bl.ocks.org

Gist | bl.ocks.org

GitHub is a powerful collaborative tool for creating software. It allows developers to work on project simultaneously, tracking new features and working together to create open source software.

Gists are a small feature of GitHub. They're mostly used for sharing and saving small snippets of useful code.

They really come in handy for us when paired with the website, bl.ocks.org.

Here's why

Bl.ocks will pull in your Gists to show you an actual embedded version of it. So we can take embed code from Datawrapper and other services, and use Gist + Bl.ocks to test them on the web.

You'll often be embedding the visuals you create in WordPress, and when you do that, you don't really need this setup. But for previewing work and showing me what you've created, this is a really helpful and easy workflow.

Choose one of the charts to use and follow these instructions.

How to use bl.ocks.org

  1. Create a new Gist.
  2. Give it a descriptive name, like Facebook IPO chart, in the Gist description field.
  3. Name the file index.html.
  4. Click Create Public Gist.
  5. Now go to bl.ocks.org/[githubusername], replacing [githubusername] with your username. My url would be bl.ocks.org/wmandrews.
  6. You should see your Gist on the page. Click the description text and it should load your code in a preview page.

There's also a few more instructions on bl.ocks, but these are the essentials.

Now do the same process with the other chart.

Chartbuilder

Chartbuilder, an open source tool from Quartz

Interactivity is a great feature of the web, but sometimes it simply isn't necessary. Every chart doesn't need the ability to roll over and see the data point. This is often overkill.

Static charts, like those that you can create with Chartbuilder, are a quick and elegant solution to simple data viz. This is a fairly new tool, so sometimes it can be a little buggy.

Chart types: Bar, column, line, scatterplot.

Google Charts

Google Charts

Google has its own tool for creating charts. They're not as elegant as the other tools we've seen, but they're much more flexible and offer a lot more visual types. It has a useful documentation, but it is a little more technical than the other tools because you have to actually edit code.

Chart types: Pie, scatter, line, bar, treemap, and many more. Select an example to alter it for your purposes.
