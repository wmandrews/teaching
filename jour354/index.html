Course description

This is a course that will explore multimedia journalism, what it means, and how it is practiced in today's newsrooms. Specifically, we will focus on interactive packages that combine several media types to create a story that is stronger and more compelling because of it.

There are many new and emerging technologies, but we will focus on creating packages using HTML, CSS and JavaScript. Using these common and well-documented languages, it is possible to create everything from a fully-custom and professional multimedia package to a responsively-designed web site. Of course, this course won't explore everything there is to know about these skills, but by the end of the semester you should have the tools to code and know your way around the web.

In addition to learning these skills, we will spend time discussing and analyzing stories from some of the leading newsrooms and media companies. We will examine how they are successful and where they can be improved.

The skills you are expected to learn include, but are not limited to:

  • Valid HTML/CSS layouts
  • Basic JavaScript
  • How to use open-source scripting libraries to make coding easier (and more fun)
  • Web usability and clear interaction design

Homework

Each week, you will come ready to discuss a multimedia interactive of your choice. Be prepared to explain why you chose the project, why you think they are effective or why you think they are ineffective. You will judge your chosen pieces based on news value, usability and aesthetic presentation. You will also choose a part of the interactive that you would like to learn how to do.

Here are some good resources for finding new and interesting visual journalism.

Also, we will be working through Code Year courses during the semester. Be prepared to show your progress and talk about what exercises challenged you.

Written assignment

Design critique: Select an multimedia package online to review and analyze. You will write a critical analysis of the project, explaining what the project does well and where its shortcomings lie. The paper should take into account information architecture, usability and overall design, as well as any bugs or programming flaws you might find.

Please cite Krug or any other sources you use to examine the project and back up your conclusions.

The paper should be double-spaced; undergrad students should write between 5-6 pages; graduate students should write between 7-8 pages.

Due date March 15.
